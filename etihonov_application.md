**[ НАЗВАНИЕ ПРОЕКТА ]**

“Система контроля обучения”

**[ ПРОБЛЕМНОЕ ПОЛЕ ]**
Преподавателям технических и естественно-научных дисциплин (физика, математика, инженерия, теория программирования) часто необходима система для проверки глубокого теоретического понимания, а не только практических навыков. Существующие LMS (как Moodle) и конструкторы тестов (Google Forms, Quizlet) имеют существенные ограничения:

1. Сложность создания сложных вопросов: Неудобно или невозможно вставлять математические формулы, фрагменты кода с подсветкой синтаксиса, чертежи или схемы непосредственно в условие вопроса и варианты ответов.

2. Жёсткость форматов: Ограниченный набор типов вопросов (часто только выбор варианта, верно/неверно), что не позволяет создавать задания на установление соответствия, свободный ввод ответа (с проверкой по шаблону), упорядочивание последовательностей - важные для технических предметов.

3. Закрытость и сложность развёртывания: Многие удобные сервисы - проприетарные или облачные, что не позволяет развернуть систему на своём сервере для полного контроля над данными и процессами.

В результате преподаватели тратят непропорционально много времени на преодоление ограничений инструментов, а не на создание качественных заданий.

**[ ЗАКАЗЧИК / ПОТЕНЦИАЛЬНАЯ АУДИТОРИЯ ]**

Заказчик

Преподаватели вузов, колледжей и курсов по техническим, математическим и естественно-научным дисциплинам. Основные пользователи. Им нужен инструмент, который понимает специфику их предмета: формулы, код, схемы. Ценят возможность создать один «умный» шаблон вопроса, который сгенерирует множество вариантов.

Методисты и авторы образовательного контента. Заинтересованы в создании переиспользуемых-банков вопросов, которые можно легко экспортировать, импортировать и комбинировать в разные курсы.

Студенты. Для них важна понятная интерпретация сложных вопросов и ясная обратная связь, а также возможность тренироваться на вариативных заданиях.

**[ АППАРАТНЫЕ / ПРОГРАММНЫЕ ТРЕБОВАНИЯ ]** 

Продукт разрабатывается под все известные операционные системы в данный момент и к моменту сдачи должен существовать в виде версий для следующих конфигураций:

* Клиент: Современные браузеры (Chrome 100+, Firefox 100+) с поддержкой JavaScript.
* Сервер для развертывания: Linux0сервер (Ubuntu 22.04 LTS / Rocky Linux 9) с Docker и Docker Compose. Минимальная конфигурация: 1-2 vCPU, 2-4 ГБ ОЗУ, 20 ГБ SSD.
    
**[ ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ ]**
Для преподавателя (создание контента):
Редактор вопросов с поддержкой гибридного форматирования: текст + Markdown + LaTeX (для формул).
Богатый набор типов вопросов:

* Один/несколько правильных ответов.

* Ввод свободного ответа (с проверкой по регулярному выражению или точному совпадению).

* Установление соответствия между элементами двух списков.

* Ранжирование/упорядочивание элементов списка.

* Ввод числового ответа (с допустимой погрешностью).

* Создание банка вопросов с тегами и сложностью.

* Генерация вариантов: Возможность создать тест, где система автоматически выберет N случайных вопросов из определённой темы или банка, создавая уникальные варианты для каждого студента.

* Импорт/экспорт вопросов в формате JSON, YAML, Markdown для обмена и резервного копирования.

* Настройка системы оценивания: Гибкая схема баллов, штрафы за попытки, различные стратегии подсчёта итога.

Для студента (прохождение теста):

* Чистый, сфокусированный интерфейс для прохождения теста.

* Поддержка отображения всех типов вопросов (формулы, код, изображения) без искажений.

* Таймер и сохранение ответов по мере ввода (автосохранение).

* Просмотр результатов с детализацией: какие ответы даны, какие - правильные, комментарии преподавателя.

Для преподавателя (аналитика):

* Детальная статистика по тесту: средний балл, распределение результатов, вопросы с наибольшим количеством ошибок.

* Анализ ответов на каждый вопрос: частотность выбора каждого варианта, анализ ответов.

* Выгрузка сводных отчетов в CSV/JSON.


**[ ПОХОЖИЕ / АНАЛОГИЧНЫЕ ПРОДУКТЫ ]**

1. Moodle Quiz. Самый прямой аналог. Очень мощный, но чрезвычайно сложный для освоения. Интерфейс редактора вопросов перегружен, работа с LaTeX требует дополнительных плагинов и настроек. Развертывание полноценного Moodle - нетривиальная задача.

2. Google Forms, Microsoft Forms. Просты для базовых опросов, но абсолютно не подходят для технических дисциплин (нет поддержки LaTeX, подсветки кода). Нет контроля над данными (облачные).

3. Quizlet, Kahoot!. Ориентированы на игровое и мобильное обучение, не поддерживают сложные форматы вопросов и серьёзную аналитику.

4. iSpring Suite, Articulate QuizMaker. Платные авторские инструменты, часто интегрируются с PowerPoint. Фокус на корпоративном обучении, развернуть свою систему на их основе нельзя.

Отличие данного проекта: Специализированная, open-source платформа, которая из коробки(батарейки входят в набор) предлагает преподавателю «технический» редактор вопросов (LaTeX + код) и гибкие настройки генерации вариантов, при этом оставаясь простой в развертывании (Docker) и использовании.

**[ ИНСТРУМЕНТЫ РАЗРАБОТКИ, ИНФОРМАЦИЯ О БД ]**

* Бэкенд: Golang + Gin.

* Фронтенд: React + Vite. Для отображения формул - KaTeX.

* БД: PostgreSQL.

* Развертывание: Docker, Docker Compose.

**[ ЭТАПЫ РАЗРАБОТКИ ]**

* Проектирование модели данных и API (2 недели): Определение структуры для хранения вопросов разных типов, тестов, результатов.

* Разработка ядра API и базовых моделей (3 недели): Реализация на Django, настройка аутентификации и ролей.

* Создание редактора вопросов (4 недели): Самый сложный этап. Разработка веб-интерфейса с переключением типов вопросов, полем для формул и вставки кода.

* Реализация механизма прохождения теста (2 недели): Интерфейс студента, логика таймера, отправки ответов.

* Разработка модуля аналитики и отчетов (2 недели): Страницы со статистикой, выгрузка данных.

* Тестирование, документация, подготовка к деплою (3 недели): Написание инструкции по развертыванию через Docker Compose, исправление ошибок.
